一、 概述

1.1 实际场景

  • 备份
  • 代码还原
  • 协同开发
  • 追溯

1.2 版本控制器的方式

  • SVN：集中式版本控制工具
  • GIT：分布是版本控制工具

1.3 工作流程图

命令：

  • clone：从远程仓库中克隆代码到本地仓库
  • checkout：从本地仓库中检出一个仓库分支然后进行修订
  • add：在提交前将代码提交到暂存区
  • commit：提交到本地仓库，本地仓库中保存修改的各个历史版本
  • fetch：从远程库，抓取到本地仓库，不进行任何的合并动作，一般操作较少
  • pull：才能够远程库拉取到本地库，自动进行合并，然后放到工作区
  • push：修改完成后，将代码推送到远程仓库（gitlab）

二、基础操作

2.1 介绍

git工作目录下对于文件的修改（增加、杀出、更新）会存在几个状态，这些修改的状态会随着我们执行git的命令而发生变化。

图示：

  • git add：工作区 —> 暂存区

  • git commit ：提交暂存区内容到本地仓库的当前分支

    • 命令形式：git commit -m “注释内容”

  • git status：查看修改的状态

  • git add：添加工作区一个或者多个文件的修改到暂存区

  • git reset –hard commitID

    • commitID可以使用git log查看

2.2 忽略列表

有一些文件无需固体管理，通常都是自动生成的日志文件、编译过程创建的临时文件。

在工作目录中创建.gitignore。

三、git远程仓库

3.1 分类

  • https://github.com：环境问题访问比较慢
  • https://gitee.com：服务器在国内访问比较快
  • https://about.gitlab.com：git私服，生产环境一般使用gitlab_jenkins配合

3.2 操作远程仓库

添加

命令：git remote add <远端名称><仓库路径>

  • 远端名称：默认是 origin
  • 仓库路径，从远端服务器获取URL，例如：https://github.com/somedaycry/git_test.git

查看

命令：git remote

**推送**到远程仓库

命令：git push [-f] [-set-upstream][远端名称[本地分支名][:远端分支名]]

  • 如果远程分支名和本地分支名称相同，则可以直系本地分支
  • 如果报错建议使用SSH KEY连接

3.3 克隆远程仓库

命令：git clone <仓库路径>[本地目录]

  • 本地目录可以省略，自动生成本地目录

3.4 拉取远程仓库

命令：git pull [remote name] [branch name]

  • 拉取指令就是将远端仓库的修改拉到本地并自动进行合并，等同于fetch+merge
  • 如果不指定远端名称和分支名，则抓取所有并更新当前分支。
git pull origin master
